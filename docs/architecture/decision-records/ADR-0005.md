# ADR-0005: Exception Hierarchy Rooted at HereException

**Status:** Accepted  
**Date:** 2026-04-20

## Context
Consumers must be able to handle HERE errors at varying granularities (catch-all vs. specific type).

## Decision
`HereException : Exception` is the base. Specializations (`HereNetworkException`, `HereAuthenticationException`, `HereRateLimitedException`, `HereInvalidRequestException`) carry type-safe properties. `HereErrorCode` enum provides a structured error code on every instance.

## Consequences
- Consumers can `catch (HereException)` for all HERE errors or catch a specific subtype.
- No platform-specific exceptions in Common (e.g. no `HttpRequestException`).
